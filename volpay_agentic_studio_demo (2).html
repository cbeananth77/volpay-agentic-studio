Choose activation mode
How do you want to go live? You can switch between modes after activation.
Shadow mode first Recommended
The agent runs against every matching live payment but applies no changes whatsoever. It records what it would have done, compares that against what your ops team actually did, and builds an accuracy report — before a single payment is touched automatically. After the shadow run, you review the accuracy data and promote to active with one click.

Shadow run will stop when 5,000 payments have been observed. You will receive a notification and accuracy report to review before deciding to promote.
Activate directly Advanced
The agent starts processing live payments immediately at your configured thresholds. Suitable if you have previously validated accuracy through shadow mode or another mechanism.
Note: We recommend shadow mode for first-time activation in a new environment.
What happens during shadow mode — step by step
1
Agent activates in read-only mode
VolPay invokes the agent on every payment matching your incidence codes — exactly as it would in active mode. The agent generates its full output: enrichment suggestion, confidence score and reasoning trace.
2
No changes applied to live payments
All agent suggestions are recorded in the Shadow Mode Dashboard but nothing is written to the payment. Your ops team handles exceptions through the standard queue, unaware the agent is observing.
3
Accuracy measured against actual outcomes
Once each payment completes — STP'd, manually repaired or returned — the system compares what the agent suggested against what actually happened. This gives ground-truth accuracy, not just internal model confidence.
4
Shadow Mode Dashboard builds in real time
The dashboard shows accuracy by incidence code and corridor, confidence distribution, divergence cases (where the agent would have been wrong), projected STP lift and estimated monthly saving at your payment volumes.
5
You review and promote when ready
When the shadow run threshold is reached you receive a notification. The system pre-populates recommended confidence thresholds based on observed accuracy. One click promotes the agent to active — with evidence behind every threshold setting.
